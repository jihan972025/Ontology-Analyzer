🔬Ontology Analyzer

AST-based code structure visualization, security scanning & LLM-assisted analysis

Ontology Analyzer는 소스 코드의 클래스, 메소드, 함수 간 관계를 시각적으로 분석하는 데스크톱 도구입니다. 폴더를 선택하면 코드를 파싱하여 호출 관계, 상속 구조, import 의존성을 그래프로 렌더링합니다.

Electron + React 프론트엔드와 Python FastAPI 백엔드로 구성되며, 모든 의존성이 번들되어 오프라인 환경에서도 설치 즉시 사용 가능합니다.

🔴Circular Dependency Detection

순환 import/호출 관계를 DFS back-edge 알고리즘으로 자동 감지합니다. 순환 참조가 발견되면 해당 엣지가 빨간색 점선으로 하이라이트되고, 관련 노드 사이의 연결선이 강조 표시됩니다.

우측 상단의 N cycles 배지를 클릭하면 순환 참조 노드를 순차적으로 탐색할 수 있습니다. 노드 클릭 시 우측 속성창에 Cycle Path(순환 경로)가 BFS 기반으로 표시됩니다.

순환 참조의 문제점

  • 무한 루프 위험
  • 단위 테스트 어려움
  • 변경 영향 확산 (ripple effect)
  • 빌드 순서 문제
  • 코드 가독성 저하 및 재사용성 감소
순환 참조 해결 방법: 인터페이스 추출(Dependency Inversion), 중재자 패턴(Mediator), 이벤트 기반 분리 등을 검토하세요.

👻Dead Code Detection

어디에서도 호출/참조되지 않는 메소드/함수를 탐지합니다. Fan-in(incoming 호출 수)이 0인 노드를 데드 코드로 판정하여 회색, 점선 테두리, 낮은 불투명도로 표시합니다.

우측 상단의 N dead 배지를 클릭하면 데드 코드 노드를 순차 탐색합니다.

Entry point(main 함수), 이벤트 핸들러, 리플렉션 호출, 외부 API 엔드포인트 등은 실제로 사용되지만 정적 분석에서 데드 코드로 오탐될 수 있습니다.

⚡Impact Analysis

특정 노드를 클릭하면 해당 노드 변경 시 영향 받는 노드들을 BFS 3단계로 탐색하여 하이라이트합니다.

1차
직접 호출 — 가장 강한 영향
2차
간접 영향
3차
파급 영향

우측 속성창의 Impact Analysis 섹션에서 각 단계별 영향 노드 목록을 확인하고 클릭하여 이동할 수 있습니다.

📊Complexity Metrics

측정 항목

MetricDescriptionCalculation
Fan-in이 노드를 호출하는 수Incoming calls/references 엣지 수
Fan-out이 노드가 호출하는 수Outgoing calls/references 엣지 수
Lines메소드 본문 라인 수{ } 매칭으로 계산

시각적 반영

  • 노드 크기 — 연결 수(connections)에 비례: 연결 많은 노드 → 큰 원
  • 노드 색상 — Fan-in + Fan-out 합산으로 히트맵:
    • 낮음 (0~2) → 클러스터 기본 색상
    • 중간 (3~5) → 주황 블렌딩
    • 높음 (6+) → 빨간/주황 강조

복잡도가 높은 노드의 문제점

  • Fan-in 높음 → 많은 곳에서 의존 → 변경 시 영향 범위 큼
  • Fan-out 높음 → 많은 것에 의존 → 하나 바뀌면 깨질 가능성 높음
  • Lines 높음 → 메소드가 너무 김 → 분리(Extract Method) 검토
  • Fan-in + Fan-out 모두 높음 → "God Method" → 리팩토링 최우선 대상
활용법: 그래프에서 크고 붉은 노드를 찾으면 복잡도 높은 핵심 코드입니다. Fan-out 10 이상이면 책임 분리, Lines 50 이상이면 메소드 분할을 검토하세요.

💡Code Suggestions

분석 완료 후 코드 품질 개선을 위한 자동 제안을 생성합니다. 우측 상단의 N tips 배지를 클릭하면 제안 패널이 열립니다.

감지 규칙 (9가지)

CategoryDetection ConditionPriority
Circular Deps순환 참조 edge 연결 컴포넌트High
SecuritySemgrep 취약점 severity별 그룹High~Low
High Fan-InFan-in ≥ 8 (medium), ≥ 12 (high)Med~High
High Fan-OutFan-out ≥ 8 (medium), ≥ 12 (high)Med~High
Large FunctionLines ≥ 40 (medium), ≥ 80 (high)Med~High
Hub NodeFan-in + Fan-out ≥ 15 (개별 < 8)Med~High
Deep Inheritanceextends chain depth ≥ 3Medium

Dead Code호출되지 않는 함수 (파일별 그룹)Low

제안 패널 사용법

  • 카테고리 필터 — 상단 탭으로 특정 카테고리만 필터링
  • 카드 확장 — 제안 카드 클릭 시 상세 설명과 관련 노드 목록이 펼쳐집니다
  • 노드 이동 — 관련 노드 클릭 시 해당 노드로 포커스 이동
  • 우선순위 색상 — 빨강 = High, 주황 = Medium, 회색 = Low
제안은 정적 분석 기반 휴리스틱이므로 프로젝트 특성에 따라 조정이 필요할 수 있습니다. 유틸리티 함수의 높은 Fan-in은 정상적일 수 있으며, 테스트 코드의 Dead Code 판정은 오탐일 수 있습니다.

🛡Vulnerability Scanning (Semgrep)

코드 보안 취약점 탐지에 Semgrep (오픈소스 AST 기반 SAST 도구)를 사용합니다. 번들된 커스텀 룰과 Semgrep 커뮤니티가 관리하는 수천 개의 보안 규칙을 적용합니다.

취약점이 있는 노드에는 빨간 점 인디케이터가 표시됩니다. 우측 상단 N vulns 배지 클릭으로 취약 노드를 순차 탐색하고, 노드 클릭 시 속성창의 Security Issues 섹션에서 상세 내용을 확인합니다.

패키지 인스톨러에 Semgrep이 번들되어 있어 별도 설치 없이 바로 사용할 수 있습니다.

🔎Node Search

Ctrl+F 또는 우측 하단 검색 버튼을 클릭하면 검색바가 열립니다. 클래스명, 메소드명을 입력하면 매칭되는 노드 목록이 표시되고, 클릭 시 해당 노드로 이동합니다.

🗺Minimap

캔버스 우하단에 전체 그래프의 축소 뷰가 표시됩니다. 파란 사각형이 현재 보이는 영역을 나타내며, 드래그하여 빠르게 다른 영역으로 이동할 수 있습니다.

💻Code Preview

노드 위에 마우스를 올리면 해당 메소드/함수의 소스 코드 일부가 툴팁으로 표시됩니다. 파일명, 라인 번호와 함께 전후 5줄의 코드를 미리 볼 수 있습니다.

🌳Inheritance Tree

우측 하단 GitBranch 버튼을 클릭하면 extends/implements 관계만 필터링하여 클래스 계층 구조를 시각화합니다. 상속 관계에 관여하는 노드만 표시됩니다.

🔗Method Trace

우측 하단 Route 버튼을 클릭하면 메소드명을 입력하여 양방향 호출 체인을 추적합니다. 선택한 메소드를 중심으로 호출자(callers)와 피호출자(callees)를 재귀적으로 탐색합니다.

트레이스 결과는 전용 레이아웃으로 표시되며, 관련된 노드와 엣지만 초록색으로 하이라이트됩니다.

📈Mermaid Diagrams

분석된 코드 구조를 3가지 다이어그램으로 내보낼 수 있습니다.

Class Diagram

상속, 구현, 메소드
Flowchart
호출 그래프
메소드/함수/클래스 간 흐름
Sequence Diagram
호출 순서
참여자 간 순차 호출

사용법

  1. 우측 하단 Mermaid 버튼 클릭
  2. 다이어그램 타입 선택: Class / Flowchart / Sequence
  3. 인터랙티브 프리뷰에서 확대/축소, 검색
  4. 파일 리스트에서 특정 파일 클릭하여 필터링
  5. Copy 버튼으로 Mermaid 코드 복사

📷PNG Export

우측 하단 다운로드 버튼을 클릭하면 현재 그래프를 PNG 이미지로 저장합니다. 캔버스에 렌더링된 상태 그대로 내보내집니다.

🤖LLM Chat

코드 분석 결과를 컨텍스트로 활용하여 AI와 대화할 수 있습니다. Chat 아이콘을 클릭하면 채팅 패널이 열리며, 그래프 요약, 선택된 노드, 취약점 정보가 자동으로 포함됩니다.

사용법

  1. Chat 아이콘 클릭으로 채팅 패널 열기
  2. Settings 아이콘으로 LLM 프로바이더 설정
  3. 코드 구조, 의존성, 보안 이슈에 대해 질문
  4. 스트리밍 응답으로 실시간 출력 확인

💬POST /api/chat

코드 분석 컨텍스트를 포함하여 LLM과 대화합니다. Server-Sent Events(SSE) 스트리밍 응답을 반환합니다.

Request

{ "messages": [ { "role": "user", "content": "Explain the circular dependencies" } ], "provider": { "provider_id": "openai", "api_key": "sk-...", "model": "gpt-4o", "endpoint_url": "https://api.openai.com/v1", "api_format": "openai" }, "context": { "graphSummary": { "totalNodes": 42, "cycleCount": 3 } }, "stream": true }

Response (SSE Stream)

data: {"content": "The circular dependencies..."} data: {"content": " involve three nodes..."} data: [DONE]
